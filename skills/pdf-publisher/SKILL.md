# PDF Publisher — PDF 출력 스킬

마크다운 형식의 이력서를 PDF 문서로 변환하는 스킬.
프로필 이미지가 제공된 경우 PDF에 포함한다.

## 실행 조건
- 최종 마크다운 이력서(`05_final_resume.md`)가 준비된 경우
- 사용자가 PDF 출력을 요청한 경우
- 파일 형식 변환(마크다운→PDF)이 필요한 경우

## 워크플로우
### Step 1: 변환 방식 선택
플랫폼에서 사용 가능한 도구에 따라 변환 방식 선택:
1. **HTML→Puppeteer/Playwright**: 마크다운을 HTML로 변환 후 브라우저로 PDF 렌더링 (가장 높은 품질)
2. **pandoc**: pandoc이 설치된 경우 `pandoc input.md -o output.pdf` (간단한 변환)
3. **Markdown → Text → weasyprint/python**: Python 기반 변환
4. **수동 가이드**: 자동 변환이 불가능한 경우 사용자에게 수동 변환 방법 안내

### Step 1-B: 프로필 이미지 처리
1. `_workspace/01_parsed_resume.json`에서 `personal_info.profile_image` 확인
2. 이미지가 있으면:
   - **로컬 파일(`file://`)** → Read 도구로 읽어서 base64 인코딩
   - **URL(`http(s)://`)** → webfetch로 다운로드 후 base64 인코딩
   - **base64 데이터** → 직접 HTML `<img>` 태그에 삽입
3. 이미지가 없으면 HTML 헤더에 이미지 영역 생략

### Step 2: PDF 생성
1. 마크다운을 HTML로 변환 (디자인 템플릿 + 프로필 이미지 반영)
2. HTML에 CSS 스타일 적용 (폰트, 색상, 여백, 페이지 설정, 이미지 크기/위치)
3. PDF로 렌더링 (적절한 페이지 크기, 여백, 폰트 임베딩)
4. `_workspace/05_final_resume.pdf`에 저장

### Step 3: 품질 확인
1. PDF 텍스트 선택 가능 여부 확인
2. 페이지 나눔이 적절한지 확인 (섹션 중간에 페이지가 나뉘지 않도록)
3. 폰트가 올바르게 렌더링되었는지 확인
4. 프로필 이미지가 올바르게 표시되는지 확인 (제공된 경우)

## 원칙
- **텍스트 선택 가능성을 최우선으로** — 이미지 기반 PDF는 ATS에서 읽을 수 없다. 반드시 텍스트 레이어가 포함된 PDF를 생성하라
- **파일 크기를 고려하라** — 폰트 임베딩 시 서브셋을 사용하여 파일 크기를 최소화하라 (5MB 이하 권장)
- **플랫폼 도구에 적응하라** — 사용 가능한 도구에 따라 변환 방식을 선택하고, 실패 시 대체 방안을 제시하라
- **변환이 불가능하면 솔직하게 안내하라** — "현재 플랫폼에서 PDF 생성을 지원하지 않습니다. 마크다운 파일을 다운로드받아 VS Code에서 'Markdown PDF' 확장으로 변환하세요"처럼 대안을 제시하라
- **프로필 이미지는 base64로 인코딩하여 HTML에 포함하라** — 외부 파일 참조는 PDF에서 작동하지 않을 수 있다

## 검증
- [ ] PDF가 생성되었는가?
- [ ] PDF에서 텍스트 선택이 가능한가?
- [ ] 디자인(폰트, 색상, 여백)이 올바르게 반영되었는가?
- [ ] 페이지 나눔이 적절한가?
- [ ] 파일 크기가 적절한가? (5MB 이하)
- [ ] 프로필 이미지가 올바르게 표시되는가? (제공된 경우)
- [ ] 이미지가 텍스트 레이어를 방해하지 않는가?

## 참고
- 마크다운→HTML 변환 시 GitHub 스타일 마크다운(GFM) 사용
- CSS `@page` 규칙으로 페이지 크기와 여백 설정
- 한글 폰트가 필요한 경우 시스템에 설치된 폰트 확인
- Puppeteer/Playwright 사용 시 `--no-sandbox` 플래그 필요할 수 있음
- 이미지 base64 인코딩 시 파일 크기가 커질 수 있으므로 1MB 이하 이미지 권장
