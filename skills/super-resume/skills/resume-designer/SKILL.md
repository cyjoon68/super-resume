# Resume Designer — 이력서 디자인 스킬

이력서 콘텐츠에 디자인 템플릿을 적용하고 시각적 완성도를 높이는 스킬.
프로필 이미지가 제공된 경우 템플릿에 이미지를 포함한다.

## 실행 조건
- 최종 이력서 콘텐츠가 준비된 경우
- 사용자가 디자인 변경을 요청한 경우 (시나리오 C)
- PDF 출력 전 시각적 포맷팅이 필요한 경우

## 워크플로우
### Step 1: 템플릿 선택
기본 템플릿 중 사용자에게 추천 (또는 사용자 선택):
1. **심플 (Classic)**: 최소한의 디자인, 흑백, 정보 전달 중심
2. **모던 (Modern)**: 세련된 컬러 악센트, 깔끔한 라인, 여백 활용
3. **크리에이티브 (Creative)**: 독특한 레이아웃, 컬러 블록, 아이콘 활용 (디자인/마케팅 직군용)
4. **ATS 최적화 (ATS-friendly)**: 심플한 구조, 키워드 가시성 극대화, 표/이미지 최소화

### Step 1-B: 프로필 이미지 처리 (제공된 경우)
1. `_workspace/01_parsed_resume.json`에서 `personal_info.profile_image` 확인
2. 이미지가 있으면 템플릿별 배치:
   - **모던:** 헤더 좌측에 원형 이미지 (직경 80px)
   - **크리에이티브:** 헤더 영역에 통합, 배경 효과와 함께
   - **심플/ATS:** 이미지 표시 안함 (텍스트 중심)
3. 마크다운에서 이미지는 표준 `![profile](path)` 형식 사용
4. HTML/PDF 변환 시 base64 인코딩 또는 파일 참조 사용

### Step 2: 콘텐츠 → 마크다운 포맷팅
1. 콘텐츠를 선택된 템플릿 스타일의 마크다운으로 구조화
2. 헤더 계층 구조 적용 (# ## ###)
3. 볼드/이탤릭/리스트를 활용한 가독성 향상
4. 구분선, 들여쓰기 등 시각적 요소 추가
5. 프로필 이미지가 있으면 헤더 영역에 이미지 태그 포함

### Step 3: 디자인 세부 튜닝 (사용자 요청 시)
사용자가 세부 조정을 요청하면:
- 폰트 변경 (serif/sans-serif, 크기 조정)
- 컬러 톤 변경 (차분한/따뜻한/차가운 톤)
- 여백 및 레이아웃 조정 (단일/2단, 여백 확대/축소)
- 특정 섹션 강조/축소
- 프로필 이미지 위치/크기/스타일 조정

### Step 4: 마크다운 출력
`_workspace/05_final_resume.md`에 저장 (프로필 이미지 포함)

## 원칙
- 템플릿은 콘텐츠를 보조하는 도구일 뿐 — 디자인이 내용을 압도하지 않게 하라
- 모든 템플릿은 ATS 호환성을 유지해야 한다 — 텍스트 기반, 선택 가능, 표준 섹션 헤더
- 템플릿 선택 시 사용자의 직군을 고려하라 — 개발자는 심플/모던, 디자이너는 크리에이티브 선호 경향
- "이 템플릿을 추천합니다"라고 제안할 때는 이유를 함께 설명하라 (직군, 업계 관행 등)
- 프로필 이미지는 있으면 좋고 없어도 무방한 옵션으로 처리하라 — 이미지가 핵심 정보를 가리지 않게 하라

## 검증
- [ ] 마크다운 형식이 유효한가?
- [ ] 헤더 계층이 올바른가? (h1 → h2 → h3 순서)
- [ ] ATS 호환성이 유지되는가?
- [ ] 시각적 일관성이 있는가? (폰트, 컬러, 여백)
- [ ] 사용자 디자인 요청이 반영되었는가?
- [ ] 프로필 이미지가 올바르게 배치되었는가? (제공된 경우)
