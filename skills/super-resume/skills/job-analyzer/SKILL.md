# Job Analyzer — 공고 분석 스킬

채용 공고 URL을 읽고 구조화된 요구사항 데이터로 변환하는 스킬.

## 실행 조건
- 사용자가 채용 공고 URL을 1개 이상 제공한 경우
- 공고 내용을 기반으로 이력서 맞춤 수정이 필요한 경우
- 어떤 기술과 경험을 강조해야 할지 분석이 필요한 경우

## 워크플로우
### Step 1: URL 병렬 Fetching
1. 각 URL에 대해 webfetch 도구로 내용 가져오기
2. fetching 실패 시 각 URL별로 1회 재시도
3. 실패한 URL은 목록에 기록하고 계속 진행

### Step 2: 공고 내용 분석
각 URL에 대해 다음 정보 추출:
- **회사 정보**: 회사명, 업종, 문화 (알 수 있으면)
- **기술 스택**: 필수 기술, 우대 기술
- **경험 요구사항**: 최소 경력, 특정 도메인 경험
- **직무 설명**: 주요 책임, 기대 역할
- **우대사항**: 추가 점수가 되는 항목
- **핵심 키워드**: ATS 매칭용 중요 키워드 목록

### Step 3: 공고 간 공통점/차이점 분석
1. 여러 공고가 있을 경우 공통 요구사항 식별
2. 공고별 특화 요구사항 식별
3. 우선순위가 높은 기술/경험 식별

### Step 4: JSON 출력
1. `_workspace/01_job_analyses.json`에 저장
2. 각 공고의 원문도 `_workspace/01_job_raw/{index}.md`에 보존

## 원칙
- URL의 신뢰성을 판단하지 말고 제공된 내용 그대로 분석하라 — 사용자가 신뢰하는 공고다
- 공고 내용이 부족하면(예: 페이지만 있고 상세 내용 없음) "분석 불가"로 표시하고 추정하지 마라
- 하나의 키워드에 과도하게 가중치를 두지 마라 — 공고의 전체 맥락을 고려하라

## 검증
- [ ] 모든 URL이 fetching되었는가? (실패 URL 별도 기록)
- [ ] 각 공고의 요구사항이 올바르게 추출되었는가?
- [ ] 공통/특화 요구사항이 구분되었는가?

## 참고
- webfetch 도구 사용. 타임아웃은 URL당 30초
- 병렬 처리: 여러 URL을 한 번에 webfetch 호출
